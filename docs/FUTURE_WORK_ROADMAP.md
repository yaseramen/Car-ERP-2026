# خطة العمل المؤجلة (للمتابعة في محادثات / جلسات لاحقة)

> هذا الملف يُحدَّث عند اتفاق مالك المشروع على مهام لاحقة. المرجع التقني للسوق: `docs/PROJECT_REFERENCE.md`.

---

## السوق والإعلانات (Marketplace)

1. **تنبيهات قبل انتهاء الإعلان**  
   إشعار المورّد قبل الحذف/الانتهاء عبر **بريد** و/أو **واتساب** و/أو **إشعار داخل النظام** (حالياً: شريط تحذير في اللوحة فقط لـ 48 ساعة).

2. **رفع الصور على تخزين المنصة**  
   بدل الاعتماد على **رابط URL** فقط: رفع إلى (مثلاً) Vercel Blob / S3، وحذف الملف فعلياً عند إلغاء الإعلان أو انتهائه لتوفير المساحة.

3. **جدولة انتهاء الإعلانات أكثر من مرة يومياً**  
   خطة Vercel **Hobby** تسمح بـ Cron **مرة يومياً** فقط (`vercel.json`). لتشغيل `/api/cron/marketplace-expiry` كل ساعات: ترقية **Pro** أو **Cron خارجي** (مثلاً cron-job.org) يستدعي المسار مع `CRON_SECRET`.

4. **لوحة سوبر أدمن للسوق (تحسينات)**  
   - تعديل أسعار/مدة الباقات من الواجهة (حالياً: تفعيل/تعطيل من الجدول؛ تعديل الحقول عبر API موجود).  
   - تقرير إيرادات إعلانات السوق منفصل في لوحة الإحصائيات.

5. **مراجعة قانونية / نصوص**  
   مراجعة نصوص التنبيه والمسؤولية مع مستشار عند الحاجة (المنصة وسيط عرض فقط).

---

## كاشير / مخزن (تم إصلاح جزء منها — متابعة إن لزم)

- إن تكررت شكاوى «المنتج غير ظاهر»: التحقق من **موزّع المبيعات** ومخزنه، و`is_active`، وفلتر القسم في الكاشير.

---

## بنية ونشر (مرجع سريع)

- فشل نشر Vercel سابقاً بسبب **Cron أكثر من يومي على Hobby** — تم ضبط جدولة يومية.  
- إن تعطل الربط GitHub ↔ Vercel: **Settings → Git → Disconnect / Connect**؛ أو مراجعة **Webhooks** في GitHub.
